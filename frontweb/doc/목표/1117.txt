### 1. 흥미로운 도입
상상해보세요. 당신은 마법의 책을 가지고 있습니다. 이 책에 이름을 쓰면, 그 사람의 정보가
나타나고, 수정하거나 지울 수도 있습니다. 이것은 바로 JSP와 DAO를 사용하여
데이터베이스에 데이터를 등록, 수정, 삭제하는 과정과 비슷합니다. JSP페이지는 마법의 책과
같이 사용자 터페이스를 제공하며, DAO는 그 뒤에서 데이터를 처리하는 마법사입니다.
이번 강의에서는 이 두 마법의 조합을 어떻게 활용하는지 배워보겠습니다.

### 2. 학습목표
 - JSP와 DAO를 활용한 데이터 등록, 수정, 삭제 프로세스 이해
 - MVC 패턴의 기본원리 파악
 - 데이터베이스 연동 기술 숙지
 - 사용자 인터페이스와 서버 사이드 로직의 분리
 
 ### 3. 주요 핵심 키워드와 개념
 - **JSP(Java Server Pages)** : 서버 사이드에서 HTML을 생성하는 자바 기반 기술
 - **DAO(Data Access Object)** : 데이터베이스의 데이터에 접근하기 위한 객체
 - **CRUD** : Create(생성), Read(읽기), Update(수정), Delete(삭제)
 - **MVC 패턴** : Model-View-Controller 의 약자, 소프트웨어 설계 패턴
 - **JDCB(Java DataBase Connectivity)** : 자바에서 데이터베이스에 접속할 수 있도록 하는 API
 
 ### 4. 생각하고 토론해볼 내용

 - MVC 패턴을 사용하는 경우의 장단점은 무엇이 있을까요?

 2. DAO의 주요 책임은 무엇인가요?
 	- a) 사용자 인터페이스 제공

 	- c) 웹 서버 관리

### 6. 실무적용예시
- **사용자 관리 시스템** : 사용자 정보 등록, 조회, 수정, 삭제 기능을 포함하는 시스템.

JSP 페이지는 사용자에ㅅ게 정보 입력 폼을 제공하고, 사용자가 입력한 정보는 DAO를 통해 데이터베이스에 저장됩니다.
또한, 사용자가 요청할 때마다 DAO는 데이터베이스에서 필요한 정보르 검색하여 JSP페이지에 표시합니다.
이런 방식으로 JSP와 DAO를 활용하여 웹 어플리케이션을 구축하면, 효율적이고 유지보수가 용이한 시스템을 만들 수 있습니다.
